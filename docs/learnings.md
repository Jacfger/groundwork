# Learnings

- **Agent skill invocation is partially advisory**: After adding "MANDATORY SKILL INVOCATION" rules with `invoke skill "name"` syntax to the bootstrap, the agent now loads `diagnose` for standard bugs (confirmed across 3 test runs). However, it still skips `interview` for small changes and features — the agent interprets "fully specified" broadly and decides the prompt contains enough detail. The `interview` skill invocation remains the weakest compliance point. Stronger enforcement may require architectural changes (e.g., a tool-level guard that checks skill was loaded before allowing implementation).
- **ACP server must run via PTY**: `opencode acp` backgrounded from bash dies silently. Must start via `pty_spawn` for the process to survive. The `acp-harness.sh` `start` command has this limitation documented.
- **Test project pollution**: ACP tests that implement features (dark mode, toggle-all) permanently modify the test project. Tests must either: (a) recreate the project before each run, or (b) use `git stash`/`git checkout` to reset between tests. Otherwise, subsequent tests find existing implementations and classify as trivial.
