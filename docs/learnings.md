# Learnings

- **Agent skill invocation is partially advisory**: After adding "MANDATORY SKILL INVOCATION" rules with `invoke skill "name"` syntax to the bootstrap, the agent loads `diagnose` for standard bugs (confirmed 3/3 runs) and `interview` for features/small changes (confirmed 2/2 runs after tightening trivial criteria). The key insight: the "Trivial small change" escape hatch was too broad — the agent re-classified everything as trivial. Fix: tightened criteria to ≤1 file, ≤10 lines, no new behavior. Multi-file changes or new user-facing behavior must go through interview.
- **ACP server must run via PTY**: `opencode acp` backgrounded from bash dies silently. Must start via `pty_spawn` for the process to survive. The `acp-harness.sh` `start` command has this limitation documented.
- **Test project pollution**: ACP tests that implement features (dark mode, toggle-all) permanently modify the test project. Tests must either: (a) recreate the project before each run, or (b) use `git stash`/`git checkout` to reset between tests. Otherwise, subsequent tests find existing implementations and classify as trivial.
